Binary Search

Description

Binary search is a searching algorithm that is capable of finding a value inside a sorted list in O(logN) time complexity. It is often quite necessary to search for a value inside huge lots of data, and binary search provides a time and memory cheap method of doing so. If a list of integers is simply kept sorted as it grows, searching an element will always have O(logN) time complexity.

An alternative to binary searching a list is maintaining a Map data structure that provides O(1) lookup time, but only on the cost of memory. This is exactly why binary search is neat!

Below are the steps for Binary Search:

  1. Input search element from user
  2. Locate the middle-most element inside the sorted list
  3. Compare the element to be found with the middle-most element
  4. If the element is the one that is being searched, return it!
  5. Otherwise, check if the element to be searched is bigger or smaller than the element it was compared to
    • If smaller, repeat steps 2, 3, 4, 5, 6 for the left sublist
    • If bigger, repeat steps 2, 3, 4, 5, 6 for the left sublist
  7. Repeat the process until the element being searched is found or if the sublist is 1-item big
  8. If the 1-item sublist does not contain the item, indicate that the item is not present!

Example Implementation

Let us search for element 2!

  1. Middle Most Element of Array: 5
  2. Comparing 2 and 5 shows that 3 < 5
  3. Re-implementing Binary Search on left part of array (element 0 to 2)
  4. Middle most element is on index (0+2)/2 = 1, which is 2
  5. Comparing 2 and 2 shows that the element was found!
  6. Returning index of element to be found: 1
